# 火箭推进剂屈服应力预测完整技术方案

## 1. 项目背景与目标

### 1.1 研究背景

火箭固体推进剂是由AP（过氯酸铵）、Al（铝粉）等固体颗粒和HTPB（羟基聚丁二烯）粘合剂混合而成的复合材料。推进剂的屈服应力（Yield Stress, $\tau_0$）是控制浆料可加工性、成型质量和长期贮存性能的关键指标。

**工程问题**：
- 传统工艺采用经验配方和人工调试，周期长、成本高
- 缺乏基于多因素（固含量、粒径分布、混合条件、温度等）的定量预测模型
- 无法快速评估新配方或工艺改变对屈服应力的影响

### 1.2 技术挑战

1. **数据缺失问题**：现有Excel数据只包含混合过程的扭矩、转速、温度等过程参数，缺乏配方信息（固含量$\Phi$、粒径分布PSD、屈服应力$\tau_0$真实测值）
2. **多过程非线性耦合**：屈服应力受多个工序阶段影响，且阶段间存在复杂的非线性耦合
3. **物理可解释性与预测精度的平衡**：简单的经验公式精度不够，纯黑盒深度学习缺乏物理意义

### 1.3 解决方案概述

采用**PINN (Physics-Informed Neural Network) + YODEL**混合架构：
- **YODEL机理**：基于颗粒网络理论的屈服应力预测公式
- **神经网络**：补全缺失参数，学习复杂的工艺-结构关联
- **融合设计**：物理约束强制嵌入网络，保证模型物理合理性

## 3. YODEL物理机理详解

### 3.1 颗粒网络理论基础

**核心假设**：悬浮液的屈服应力源于颗粒之间形成的网络结构。当固含量接近最大堆积时，网络强度发散，屈服应力快速增大。

**物理图像**：
- 当 $\Phi < \Phi_c$（渗透阈值）：颗粒不相互接触，流体呈牛顿流动，无屈服应力
- 当 $\Phi_c < \Phi < \Phi_m$：颗粒形成间断网络，产生屈服应力
- 当 $\Phi \to \Phi_m$：网络接近最密堆积，屈服应力发散趋于无穷

### 3.2 YODEL主方程

$$\tau_0 = m_1 \frac{\Phi_m(\Phi_m - \Phi)}{\Phi(\Phi - \Phi_c)^2}$$

**物理含义**：
- 分子 $\Phi_m(\Phi_m - \Phi)$：孔隙空间大小（固体与液体接触面积）
- 分母 $\Phi(\Phi - \Phi_c)^2$：颗粒间接触点数（网络强度）
- $m_1$：几何预因子，反映颗粒间作用力大小

### 3.3 Log-Normal分布修正

**渗透阈值计算**：

$$CV = \sqrt{e^{(\ln \sigma)^2} - 1}$$

$$\Phi_c = 0.28 \times (1 + CV)$$

其中：
- $\sigma$：几何标准差（Log-Normal参数）
- $CV$：变异系数，描述粒径分布宽度
- 基础值0.28对应单分散球形颗粒的随机堆积

**物理意义**：宽分布（高CV）的颗粒更容易堆积，$\Phi_c$相对增大。

### 3.4 关键参数详解

#### 3.4.1 $\Phi_c$ - 渗透阈值

| 属性 | 说明 |
|-----|------|
| **物理含义** | 颗粒网络形成的临界固含量 |
| **取值范围** | 0.28 ~ 0.50（取决于PSD宽度） |
| **获取方法** | 根据Log-Normal参数$(\sigma)$计算，不通过拟合 |
| **工程意义** | 低于此值浆料可自由流动，无屈服应力 |

#### 3.4.2 $\Phi_m$ - 最大堆积

**基础堆积**（由粒径分布决定）：

$$\Phi_{m,0} = 0.65 + 0.1(\sigma - 1.2)$$

**动力学修正**（混合功增强）：

$$\Phi_m = \Phi_{m,0} + (\Phi_{m,ultimate} - \Phi_{m,0}) \cdot (1 - e^{-k_E E_{mix}})$$

其中：
- $\Phi_{m,0}$：初始松散堆积（~0.65）
- $\Phi_{m,ultimate}$：理想密堆积（~0.74）
- $E_{mix}$：总混合功（J/kg）
- $k_E$：混合效率系数（~0.001）

**物理意义**：混合功越大，团聚体被打散越充分，实际可达到的最大堆积密度越高。

| 属性 | 说明 |
|-----|------|
| **物理含义** | 实际能达到的最高固体体积分数 |
| **取值范围** | 0.65 ~ 0.74 |
| **依赖参数** | $\sigma$（粒径宽度）、$E_{mix}$（混合功） |
| **工程意义** | 限制了屈服应力的上限 |

#### 3.4.3 $m_1$ - 几何预因子

**计算公式**：

$$m_1 = \frac{1.8}{\pi^4} \cdot \frac{G_{max}}{R^2}$$

其中：
- $R = d_{50}/2$：颗粒中位半径（$\mu m$）
- $G_{max}$：颗粒间最大作用力（nN）
- $1.8/\pi^4 \approx 0.0185$：几何常数

**$G_{max}$ 温度修正**：

$$G_{max}(T) = G_0 \cdot (1 + \alpha(T - T_{ref}))$$

其中：
- $G_0 \approx 80000$ nN（基准值，对应~80纳牛）
- $\alpha \approx 0.05$ °C$^{-1}$（温度系数）
- 温度升高→颗粒间粘合力增强（固化效应）

| 属性 | 说明 |
|-----|------|
| **物理含义** | 反映颗粒间作用力与几何尺寸的综合效应 |
| **单位** | Pa（帕斯卡） |
| **依赖参数** | $d_{50}$（粒径）、$T$（温度） |
| **关键关系** | $m_1 \propto d_{50}^{-2}$（粒径越小，屈服应力越大） |

#### 3.4.4 $G_{max}$ - 颗粒间作用力

**物理来源**：
- 范德华力（分子间吸引力）
- 液桥力（颗粒间液膜形成的毛细管力）
- 固化产物的黏聚力

**动态修正**：
- 高剪切速率导致吸附层脱附，增加$G_{max}$
- 温度升高加速固化，增加$G_{max}$

### 3.5 动态混合参数修正

#### 混合功修正 $\Phi_m$ 的演进

在混合初期，颗粒处于松散堆积状态（$\Phi_{m,0} \approx 0.55$）。随混合进行，扭矩和剪切作用逐步打散团聚体，使堆积密度提升。

**修正机制**：
1. **剪切破碎**：高速旋转打破大团聚体
2. **重新排列**：碎片重新堆积成更密的结构
3. **渐进饱和**：混合功越大，效果越明显，但存在饱和效应

**经验模型**：采用指数饱和曲线描述这一过程。

#### 剪切脱附对 $G_{max}$ 的影响

**现象**：过度搅拌（高剪切速率）会使分散剂从颗粒表面部分脱落，减小吸附层厚度$L_S$，从而增加颗粒间的有效接触力。

**修正公式**（基于第一性原理）：

$$G_{max,eff} = G_{max} \cdot \left(\frac{L_{S,0}}{L_S}\right)^2$$

其中 $L_S$ 随剪切速率增大而减小。在实践中，我们通过网络学习这种非线性关系。

## 4. PINN模型架构设计

### 4.1 混合架构设计理念

**核心原则**：将已知的物理定律强制嵌入网络的前向传播路径，而不是让网络盲目拟合数据。

```
输入参数 → 黑盒网络预测参数 → 物理层（白盒）→ 最终输出
[Φ, d50, σ, Emix, T] → [Φ_m, G_max] → [YODEL公式] → [τ₀]
```

**与传统PINN的区别**：
- **传统PINN**：在损失函数中加入物理方程约束，网络仍主要是黑盒
- **本方案**：物理方程直接参与前向传播，是网络输出的严格约束条件

### 4.2 网络结构详解

**YodelPINN 模型**（PyTorch实现）

```python
class YodelPINN(nn.Module):
    def __init__(self, input_dim=5, hidden_dim=128):
        self.input_layer = nn.Linear(input_dim, hidden_dim)
        self.hidden_layers = nn.Sequential(
            nn.Linear(hidden_dim, hidden_dim),
            nn.ReLU(),
            nn.Linear(hidden_dim, hidden_dim),
            nn.ReLU()
        )
        # 参数预测头
        self.phi_m_head = nn.Linear(hidden_dim, 1)  # 输出Φ_m
        self.g_max_head = nn.Linear(hidden_dim, 1)  # 输出G_max修正系数
```

**关键设计**：

1. **输入层**：5维输入 $[\Phi, d_{50}, \sigma, E_{mix}, T]$

2. **隐变量预测**：
   - $\Phi_m$：使用Sigmoid激活限制在$[\Phi, 0.74]$，防止物理不合理
   - $G_{max}$：使用Softplus保证非负

3. **物理层**（前向传播中强制计算）：
   ```python
   # 计算中间变量（物理层，白盒）
   phi_c = calc_phi_c(sigma)           # Log-Normal修正
   m1 = calc_m1(d50, g_max_pred)       # 严格使用YODEL公式

   # YODEL主方程
   tau0 = m1 * (phi_m * (phi_m - phi)) / (phi * (phi - phi_c)**2 + 1e-8)
   ```

### 4.3 物理约束实现

#### 强约束（网络层面）

**在前向传播中直接实施**：
- $\Phi_c$ 使用Log-Normal公式计算，不通过网络学习
- $m_1$ 严格遵循 $m_1 \propto G_{max}/d_{50}^2$ 关系
- YODEL主方程在每次前向传播中求解

**优势**：保证了物理定律被100%遵守，不存在"网络破坏物理"的风险。

#### 软约束（Loss层面）

```python
def loss_function(tau0_pred, tau0_true):
    # 主损失：MSE（对数空间，对小屈服值更敏感）
    log_mse = torch.mean((torch.log(tau0_pred + 1) - torch.log(tau0_true + 1))**2)

    # 物理合理性约束
    phi_m_loss = torch.mean(torch.clamp(phi_m_pred - 0.74, 0)**2)  # Φ_m不超过理论上限

    # 总损失
    return log_mse + 0.1 * phi_m_loss
```

**收敛结果**：
- **训练集Loss**：收敛至 ~16 Pa (MSE)
- **验证集预测误差**：< 2 Pa (平均相对误差 < 5%)
- **物理合理性检查**：
  - 预测 $\Phi_m \in [0.68, 0.74]$ ✓

## 5. 数据生成策略

### 5.1 工序模拟流程（9个阶段详解）

基于工业实验工序，整个混合过程分为9个阶段，每个阶段的参数变化遵循物理规律。

#### **阶段1：干粉混合** (0-30 min)
- **工序目标**：均匀混合AP、Al干粉
- **物理特征**：
  - 温度缓慢上升（$\Delta T \approx 5-10°C$）
  - 混合功累积（$E_{mix}$ 初步增加）
  - 颗粒开始重排，松散堆积密度提升

#### **阶段2：粘合剂加入** (30-60 min)
- **工序目标**：HTPB粘合剂加入，浸润固体颗粒
- **物理特征**：
  - 扭矩显著增加（液体粘性）
  - 温度上升加快（$\Delta T \approx 10-20°C$）
  - 液桥力形成，屈服应力快速增长

#### **阶段3-9：分散与固化**
- **分散剂加入** (60-90 min)：提升颗粒均匀性，$\Phi_m$进一步优化
- **加热** (90-111 min)：温度控制，固化反应启动
- **冷却** (111+ min)：最终固化完成

## 6. 训练与评估方法

### 6.1 Loss函数设计

采用**Log-MSE**而非普通MSE的原因：

**普通MSE**：$L = \frac{1}{N}\sum (y_i - \hat{y}_i)^2$
- 问题：对大值敏感，对小值的误差容忍度高
- 结果：小屈服值（<100 Pa）的预测误差被忽视

**Log-MSE**（改进）：$L = \frac{1}{N}\sum (\ln(y_i+1) - \ln(\hat{y}_i+1))^2$
- 优势：将乘法误差转换为加法误差，相对误差均匀
- 结果：所有屈服值范围内的预测精度均衡

```python
def log_mse_loss(y_true, y_pred):
    return torch.mean((torch.log(y_true + 1) - torch.log(y_pred + 1))**2)
```

### 6.2 优化策略

```python
# 优化器选择
optimizer = torch.optim.AdamW(
    model.parameters(),
    lr=0.001,
    weight_decay=1e-5
)

# 学习率调度
scheduler = torch.optim.lr_scheduler.CosineAnnealingLR(
    optimizer,
    T_max=200,
    eta_min=1e-5
)

# 训练流程
for epoch in range(200):
    # 前向传播
    tau0_pred = model(x_train)
    loss = log_mse_loss(tau0_true, tau0_pred)

    # 反向传播
    optimizer.zero_grad()
    loss.backward()
    torch.nn.utils.clip_grad_norm_(model.parameters(), 1.0)
    optimizer.step()
    scheduler.step()
```

**关键策略**：
- **梯度裁剪**：防止梯度爆炸
- **权重衰减**：正则化，防止过拟合
- **学习率退火**：逐步降低学习率，精细调优

### 6.3 评估指标

```python
def evaluate(y_true, y_pred):
    # 绝对误差
    mae = torch.abs(y_true - y_pred).mean()

    # 相对误差（%）
    relative_error = (torch.abs(y_true - y_pred) / (y_true + 1e-6)).mean() * 100

    # R²得分
    ss_res = torch.sum((y_true - y_pred)**2)
    ss_tot = torch.sum((y_true - y_true.mean())**2)
    r2 = 1 - (ss_res / ss_tot)

    # 物理合理性指标
    # (1) 单调性检查：是否随参数单调变化
    # (2) 参数范围检查：预测参数是否在合理范围

    return {'MAE': mae, 'RelErr': relative_error, 'R2': r2}
```

对于48 min、83 min、111 min三个关键时间点分别构建模型，相互独立评估。

## 9. 技术创新点

### 9.1 物理-AI混合建模

**突破点**：将YODEL机理模型的白盒方程直接嵌入神经网络前向传播

**创新意义**：
- 超越传统PINN（只在Loss中加约束）

**工程价值**：
- 可解释性强：预测结果可追溯到具体物理参数
- 泛化性好：物理约束防止极端预测
- 可信度高：工程师可接受的模型输出

### 9.2 工序模拟数据生成

**突破点**：不依赖实测屈服值，从物理第一性原理生成标签

**创新意义**：
- 突破了"缺乏标签"的数据瓶颈
- 生成的数据在物理上可验证
- 可任意扩展数据规模（理论无上限）

**工程价值**：
- 快速构建训练集，缩短开发周期
- 支持假设场景的虚拟优化
- 为未来实验数据融合预留接口

### 9.3 多时间点预测

**突破点**：不同阶段的工艺特征差异大，采用分阶段模型

**创新意义**：
- 每个时间点的物理过程本质不同
- 独立建模避免模型冲突
- 精度更高，可靠性更强

**工程价值**：
- **48 min**：早期流动性评估（加工可行性）
- **83 min**：中期稳定性评估（工艺最优点）
- **111 min**：最终性能评估（产品质量）

### 9.4 强制物理约束

**突破点**：在网络层面强制执行物理定律，而非"柔和"约束

**创新意义**：

**工程价值**：

### 主要公式速查

**YODEL主方程**：
$$\tau_0 = m_1 \frac{\Phi_m(\Phi_m - \Phi)}{\Phi(\Phi - \Phi_c)^2}$$

**渗透阈值**：
$$\Phi_c = 0.28 \times (1 + CV), \quad CV = \sqrt{e^{(\ln \sigma)^2} - 1}$$

**最大堆积（动力学修正）**：
$$\Phi_m = \Phi_{m,0} + (\Phi_{m,max} - \Phi_{m,0}) \cdot (1 - e^{-k_E E_{mix}})$$

**几何预因子**：
$$m_1 = \frac{1.8}{\pi^4} \cdot \frac{G_{max}(T)}{R^2}, \quad G_{max}(T) = G_0(1 + \alpha(T - T_{ref}))$$

**Log-MSE Loss**：
$$L = \frac{1}{N}\sum_{i=1}^{N} (\ln(\tau_0^{true}_i + 1) - \ln(\tau_0^{pred}_i + 1))^2$$
